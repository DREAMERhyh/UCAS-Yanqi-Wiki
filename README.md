# 🦢 UCAS 雁栖湖校园指南

> 一份由国科大学子共建的、鲜活的雁栖湖校区生存与生活指南。

## 🌟 项目背景与意义

在雁栖湖校区，面对漫长的冬季、分散的园区和丰富的校园生活，我们常常遇到这样的问题：
- **信息碎片化**：哪里的麻辣香锅最正宗？哪个澡堂水最稳？选课有什么避坑指南？这些信息往往散落在各个QQ群、微信群、朋友圈和口口相传中，难以沉淀。
- **缺乏真实反馈**：官方的介绍只有客观数据，缺少真实的使用体验。

本项目旨在打造一个**结构化、可持续更新、客观信息与主观评价并存**的开源校园知识库。它不仅是一个静态网站，更是一个通过 CMS 系统支持所有同学轻松提交测评、共同维护的社区智库。

## 🗂️ 目录结构设计原则

为了兼顾"人类的物理活动直觉"与"无限扩展的品类"，本项目的目录结构遵循以下核心原则：

1. **物理实体优先**：目录层级按现实中的物理位置划分（如：东区三食堂），而非逻辑品类（如：饮品、夜宵）。因为同学们的觅食活动总是按地点发生的。
2. **标签解决品类**：具体的品类（正餐、小吃、甜点、夜宵）通过 Markdown 的 Frontmatter 标签来实现，前端通过标签筛选，打破了物理目录的局限性。
3. **跨区域品牌提级**：像"自动售货机"、"咖啡角"这种跨越物理位置、标准统一的品牌/设施，单独提级到"跨区域品牌"目录下，避免冗余。
4. **客观与主观分离**：每个品类的客观信息（价格、营业时间等）存放在 `index.md` 的 Frontmatter 中，主观评价存放在独立的 `reviews/` 目录下，避免多人编辑冲突。

## 🧩 核心功能

### 1. 动态侧边栏导航
- 自动扫描 `docs/饮食/` 目录结构
- 智能生成可折叠的侧边栏菜单
- 自动过滤 `reviews` 目录，保持导航简洁

### 2. 用户评价系统
- 支持多人同时提交评价，无冲突
- 评价按时间倒序展示
- 包含评分、甜度选择、冰量选择等字段

### 3. Decap CMS 在线投稿
- **档口信息维护**：修改咖啡角儿档口的客观信息
- **新增品类**：在咖啡角儿下新建品种文件夹
- **提交评价**：选择品类提交用户评价，自动生成带时间戳的文件

## 🔗 CMS 后台访问

本地开发时，点击顶部导航栏的【内容投稿】按钮，或直接访问 `/admin/` 路径即可进入 CMS 后台。

> **注意**：本地开发环境仅用于验证 UI 界面，完整功能需要部署到 GitHub Pages 并配置 GitHub Gateway 认证。

## 🛠️ 技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| 前端框架 | [VitePress](https://vitepress.dev/) | 基于 Vue 的极速静态站点生成器 |
| 内容管理 | [Decap CMS](https://decapcms.org/) | 无代码表单提交，原 Netlify CMS |
| 自动部署 | GitHub Actions | Push 代码自动构建发布至 GitHub Pages |
| 评价聚合 | Vue 组件 | 动态加载并展示用户评价 |

## 📝 贡献指南

### 添加新品类
1. 在对应档口目录下新建文件夹（如 `咖啡角儿/新品名/`）
2. 创建 `index.md` 文件，填写 Frontmatter 数据
3. 创建 `reviews/` 目录存放用户评价

### 修改档口信息
- 直接编辑对应目录下的 `index.md` 文件的 Frontmatter

### 提交评价
- 通过 CMS 后台提交（推荐）
- 或手动在 `reviews/` 目录下创建 `YYYYMMDD-作者.md` 文件
