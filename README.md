# 🦢 UCAS 雁栖湖校园指南

> 一份由国科大学子共建的、鲜活的雁栖湖校区生存与生活指南。

## 🌟 项目背景与意义

在雁栖湖校区，面对漫长的冬季、分散的园区和丰富的校园生活，我们常常遇到这样的问题：
- **信息碎片化**：哪里的麻辣香锅最正宗？哪个澡堂水最稳？选课有什么避坑指南？这些信息往往散落在各个QQ群、微信群、朋友圈和口口相传中，难以沉淀。
- **缺乏真实反馈**：官方的介绍只有客观数据，缺少真实的使用体验。

本项目旨在打造一个**结构化、可持续更新、客观信息与主观评价并存**的开源校园知识库。它不仅是一个静态网站，更是一个通过 CMS 系统支持所有同学轻松提交测评、共同维护的社区智库。

## 🗂️ 目录结构设计原则

为了兼顾“人类的物理活动直觉”与“无限扩展的品类”，本项目的目录结构遵循以下核心原则：

1. **物理实体优先**：目录层级按现实中的物理位置划分（如：东区三食堂），而非逻辑品类（如：饮品、夜宵）。因为同学们的觅食活动总是按地点发生的。
2. **标签解决品类**：具体的品类（正餐、小吃、甜点、夜宵）通过 Markdown 的 Frontmatter 标签来实现，前端通过标签筛选，打破了物理目录的局限性。
3. **跨区域品牌提级**：像“自动售货机”、“咖啡角”这种跨越物理位置、标准统一的品牌/设施，单独提级到“跨区域品牌”目录下，避免冗余。

## 🧩 模块化与可扩展性

**本项目的架构设计为“搭积木”模式，目录是可以随时按需增添的，无需重构现有结构。**

- **新增一个档口**：只需在对应食堂目录下新建一个以档口命名的文件夹（如 `饮食/东区三食堂/黄焖鸡/`），放入 `index.md` 即可。
- **新增一个品类**：无需修改目录！只需在测评的 Markdown 中添加对应的 `tags`（如 `[夜宵, 炸串]`），前端会自动聚合生成品类页面。
- **新增一个区域**：如果学校新建了食堂或设施，只需在对应的一级/二级目录下新建文件夹，并在 VitePress 侧边栏配置中补上一行路径即可。
- **新增一个板块**：如果未来需要增加“社团活动”或“租房指南”，只需在 `docs/` 下新建对应的顶级文件夹，完全不影响现有的四大板块。

## 🛠️ 技术栈

- **前端框架**：[VitePress](https://vitepress.dev/) (基于 Vue，极速的静态站点生成)
- **内容管理**：[Decap CMS](https://decapcms.org/) (原 Netlify CMS，提供无代码表单提交测评)
- **自动部署**：GitHub Actions (Push 代码自动构建并发布至 GitHub Pages)
